🤖 ¿Cómo funciona GitHub Copilot?

GitHub Copilot analiza el código que escribes y el contexto cercano en tu archivo o proyecto. Utiliza un modelo llamado Codex (basado en GPT) para generar sugerencias que aparecen justo en tu editor.

🧠 Flujo general de funcionamiento

graph TD
    A[🧑‍💻 Escribes código] --> B[🧠 El editor analiza el contexto]
    B --> C[☁️ Se envía el contexto a Codex en GitHub]
    C --> D[⚙️ Codex genera sugerencias relevantes]
    D --> E[💡 Aparecen sugerencias en VS Code]
    E --> F{Aceptar, editar o ignorar}

Este diagrama representa cómo GitHub Copilot transforma el contexto del editor en sugerencias en tiempo real.

💬 Tips de uso para Java Developers

  • Comienza tus métodos con comentarios tipo // Validar que un email sea válido y deja que Copilot complete el código.
  • Puedes usar // Generate DAO for Cliente y te sugerirá la clase.
  • Funciona mejor en inglés: usa prompts como // Check if user is authenticated.

🛠️ Recomendaciones

  • Revisa el código sugerido siempre. Copilot puede fallar con validaciones, seguridad o prácticas específicas.
  • Úsalo como asistente, no como reemplazo. Tú tienes la última palabra como developer.

⚙️ Cómo activar Mermaid en Hugo LoveIt

En tu config.toml agrega:

[markup]
  [markup.goldmark]
    [markup.goldmark.renderer]
      unsafe = true

[params]
  [params.page]
    mermaid = true

Esto permite que el diagrama se renderice correctamente.
